- 2026-06-30: 设计 Java 静态工厂时，不能只看静态方法和实例方法参数是否不同；还要检查 `ClassName.method(args)` 的重载解析是否会命中同名实例方法并导致“无法从静态上下文引用非静态方法”。同名静态入口若与现有实例重载冲突，应改用 `of` 等不冲突命名。
- 2026-06-30: 评估“哪些类该加静态创建方法”时，应优先检查高频业务入口和值对象；分页类 `Page` 这类常用入参应排在 `FieldReference`/`ValueReference` 等低频辅助对象之前，不要只沿着当前讨论链路遗漏更常用 API。
- 2026-06-30: 当用户要求“放到某个现有工具入口里实现”时，应优先内聚到该入口或私有辅助逻辑中，不要主动新增公开静态方法扩 API 面，除非用户明确认可这个新入口。
